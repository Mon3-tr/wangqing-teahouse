# 选择系统说明

## 核心理念

根据设计文档，本项目的选择系统遵循一个重要原则：

**"所有交互只改变「叙事的视角/心境/氛围」，不改变事件本身"**

这意味着：
- ✅ 选择会影响后续内容的呈现方式
- ✅ 不同选择会看到不同的内心独白和视角描述
- ❌ 选择不会改变故事的主线情节
- ❌ 选择不会改变故事的结局

## 如何运作

### 1. 用户做出选择

当用户遇到抉择节点时，可以选择不同的视角：

例如第一个抉择：
- **选项A**：法海的职责 - 降妨是使命，不容质疑
- **选项B**：法海的困惑 - 这真的是妖气吗？

### 2. 后续内容根据选择显示

选择之后，会出现**只有该视角才能看到的内容**：

**选择A（职责视角）会看到：**
```
法海（内心）：
不能动摇！
我是法海，金山寺住持，降妖除魔是我的天职。
无论这妖气多么温和，妖就是妖，必须除去！
```

**选择B（困惑视角）会看到：**
```
法海（内心）：
这真的是妖气吗？
我修行三十载，从未感受过如此……柔和的气息。
难道，妖也能有善念？
```

### 3. 视觉提示

当显示条件内容时，会在文字卡片上方显示一个小标签：

```
🔍 你的视角：法海的职责 - 降妖是使命，不容质疑
```

这让用户清楚地知道：这段内容是因为我的选择而出现的。

## 技术实现

### 数据结构

每个文本段可以设置条件：

```typescript
{
  type: 'text',
  speaker: '法海',
  textType: 'inner',
  condition: {
    choiceKey: 'choice-6',      // 第6个抉择节点
    requiredValue: 'duty'        // 需要选择了 'duty'
  },
  texts: ['这段内容只有选择"职责"视角才能看到']
}
```

### 渲染逻辑

```typescript
// 检查条件
if (section.condition) {
  const userChoice = userChoices[section.condition.choiceKey];
  if (userChoice !== section.condition.requiredValue) {
    return null; // 不满足条件，不渲染
  }
}
```

## 当前章节的选择结构

### 白蛇·法海篇

**第一次抉择**（索引6）- 如何看待妖气
- 选项：职责(duty) vs 困惑(doubt)
- 影响：下一段法海的内心独白

**第二次抉择**（索引11）- 如何理解白素贞的行为
- 选项：伪装(deceive) vs 真心(genuine)
- 影响：法海对白素贞行为的解读

**第三次抉择**（索引19）- 最终的内心态度
- 选项：坚守天道(firm) vs 内心挣扎(waver)
- 影响：法海的内心挣扎程度和结尾感悟

## 设计哲学

这种设计让用户成为"选择从何种视角看故事的人"，而不是"决定故事走向的人"。

无论选择什么：
- 法海都会镇压白素贞（故事结局不变）
- 白素贞都会被囚禁于雷峰塔（事件不变）
- 但用户会看到法海内心不同层次的挣扎（视角改变）

这正是本项目的核心价值："原来如此"的反差震撼 - 让用户从不同视角重新理解熟知的故事。

## 扩展建议

未来可以添加：
1. **收集册**：记录用户所有选择的视角
2. **二刷提示**：鼓励用户用不同视角重新阅读
3. **视角对比**：显示两种视角的差异
4. **隐藏彩蛋**：某些特定选择组合解锁额外内容
