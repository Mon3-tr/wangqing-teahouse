## 这个功能是什么

*一句话定义**：
这是一个**互动叙事文化网站**，借鉴视觉小说的对话、抉择、视角切换等交互形式，让用户在沉浸式重走民间爱情故事的同时收获文化知识，发出"原来耳熟能详的故事还有这样一面"的感叹。

**产品形态明确**：
- ✅ 是：以网页为载体的**互动叙事 + 文化科普**作品，借鉴视觉小说的「对话框 + 插画 + 抉择」交互来增强沉浸感
- ❌ 不是：完整的视觉小说游戏（不需要复杂存档读档、成就墙、好感度数值等游戏化重模块）
- ❌ 也不是：可以被用户“重写”的交互故事——所有交互只影响「叙事的呈现方式」（视角/心境/氛围），**不改变事件本身**
- 🎯 目的：让用户在 **15-20 分钟内**像"读了一篇会动的图文故事"那样完整体验一段民间故事，并自然吸收文化知识

**素材形式明确**：
- 画面主要采用**传统古画素材**（画像石拓片、敦煌壁画、宋元纸本设色、木版年画、连环画等）
- 古画以**静态呈现**为主，只做温柔的入场/淡入淡出/镜头推拉/文字浮现等克制过渡；**不做抖动、表情切换、后退、转身等演出动作**
- 古画本身是主体表达，交互与动效服务于“呈现古画”，而非“让古画表演”

**呈现形态明确（数字绘本长卷）**：
- 章节采用**垂直长卷滚动式**呈现，取古画长卷（《清明上河图》《千里江山图》）随卷轴展开阅读的意象——一章从头到尾就是**一道可以一直下拉的卷轴**
- 不是点一下跳一页的幻灯片交互，而是随滚轮/上拉逐段推进的阅读体验
- 古画与文字交错排布；画面随滚动进入/退出可视区时触发轻微动画（淡入、Ken Burns、卷轴展开）
- 交互分为两类、分别处理：
  - **卷上亮点（可点可不点）**：如文化彩蛋、人物头像点击查看——以卷轴中的亮点嵌入，不拦截阅读节奏
  - **卷中拦轴（必须交互）**：如抉择、视角切换点——滚动到该节点会被软拦截，画面原位出现交互元素，用户选择后才能继续下拉
- 在这个意义上，项目形态更接近**带交互的数字互动长卷绘本**，而非 Ren'Py / Galgame 那种点击翻页的传统 VN 形式

**核心创作原则（不可妥协）**：
1. **尊重原典**：故事主干、关键情节、结局走向必须忠实于原典文献（《白蛇传》以方成培《雷峰塔》传奇为底本，《牛郎织女》以《古诗十九首》《风俗通义》《小说》等为底本）
2. **完整叙述**：每个章节必须分段完整讲述故事全程，从起因 → 发展 → 高潮 → 结局，而非只截取一段抉择
3. **不重写故事**：这是“用新颖方式讲传统故事”，不是“重新创造故事结局”。所有事件、人物关系、结局都与原典一致，变的只是呈现方式
4. **交互服务沉浸**：抉择、视角切换、场景探索只影响「讲述的视角/心境/氛围」，**不改变事件本身**。用户不是“决定故事走向的人”，而是“选择从何种视角看这个故事的人”
5. **玩中学**：每个关键节点附带可选的"文献注解/文化背景"浮窗，让用户在沉浸的同时自然获得文化知识
6. **触发"原来如此"**：通过反派视角、物品视角、内心独白等方式，让熟知情节呈现出未被讲述的一面，但不能颠覆原典价值观
7. **轻量克制**：交互形式服务叙事，不堆砌游戏机制；用户应能像"看一篇会动的图文长文"那样轻松进入

*设计效果**：
1. 以原典开篇切入即被沉浸——熟悉的开场（如「金山寺，子时」「七仙女下凡沐浴」），但视角是"未被讲述的那个人"
2. 完整看完一段民间故事——从起因 → 发展 → 高潮 → 结局，比看百度百科更清楚故事来龙去脉
3. 看到结局时会说"原来法海/王母娘娘也有他的难处"——但不会觉得"原典是错的"
4. 顺便吸收一批文化知识——如"七夕原本是乞巧节、不是情人节"、"《雷峰塔》里水漫金山原本无白蛇之子"
5. 看完后产生重读冲动——为看其他视角、收集全部文化彩蛋、感受不同心境结局
6. 看完会想截图发小红书——某句台词、某张插画、某个文献注解有社交分享价值

**这次不做**：
- 颠覆原典的"魔改"剧情（如把白蛇变成反派、让法海娶白娘子）
- 全量民间爱情故事章节（比赛只做深 1-2 章 + 短篇预告占位）
- **完整的视觉小说游戏机制**——不做复杂存档/读档系统、快进、成就墙、好感度数值等游戏化重模块（仅保留克制的自动播放，便于用户专注欣赏古画）
- 战斗 / 养成 / 数值系统（与叙事核心冲突）
- 用户 UGC 剧本上传（审核成本高）
- 配音（比赛周期内成本高，先用 BGM + 音效托底）

### 三、长卷中的交互节点（嵌入卷轴中的两类交互）

**A. 卷上亮点（可点可不点，不拦截阅读节奏）**：
- **文化彩蛋亮点**：古画中的重要物件/场景出现柔光呼吸，点击弹出文化注解卡片
- **人物头像亮点**：卷中出现人物头像可点击查看人物背景/心声
- **「如果当时」幻象插段**：在原典高潮点自动插入一段幻象画面，读完随滚动自然退出、回归原典
- **点击过的亮点会变金边**并计入「文化收集册」

**B. 卷中拦轴（必须交互才能继续下拉）**：
- **抉择拦轴**：画面软锁定，中央出现 2-3 个选项卡片（“你选择从…看”），选择后锁定解除
- **视角切换拦轴**：画面软锁定，出现角色头像选项，点击后同一事件以另一视角重述
- **信物探案拦轴**（仅探案模式章节）：画面软锁定，要求在画面中点亮足够信物后才能继续
- **幕末上拉拦轴**（心境拦轴）：仅出现于末幕，让用户选择主角此刻的心境，决定后续旁白/氛围调性

**C. 边栏辅助成分（不占卷轴主体区）**：
- **侧边幕表目录**：鼠标移到页面左侧边缘浮出，可跳转到其他幕（仅限已读过的幕）
- **右下角控件**：自动播放/设置/文化收集册/退出，鼠标静止后自动隐藏

## 用户路径

## 主流程：首次玩《白蛇·法海篇》（完整 7 幕，约 20 分钟）

1. **进入首页** → 全屏 Hero 立绘 + 「这一次，由你来选择她们的命运」标题 → 用户被勾住
2. **点击「法海篇」章节卡片** → 章节封面页（角色简介 + 视角说明：「玩家将扮演法海」+ 7 幕索引预览）→ 「开始」按钮
3. **第一幕「金山初感」** → 黑场旁白「金山寺，子时。一股妖气直冲云霄……」→ 背景淡入 → 法海立绘登场 → 完整对白叙述法海感应妖气的来龙去脉 → 期间触发**文化彩蛋**「金钵来历」（可点可不点） → 幕末**抉择①**：「即刻下山降妖 / 静观七日再说」
4. **第二幕「西湖初遇」** → 完整呈现「游湖借伞」原典桥段（白蛇视角）→ 中段触发**视角切换①**「想看看法海眼中的西湖此刻是什么样？」 → 切换后看到白蛇身上的妖气如何被法海感知 → 切回主视角推进 → 触发彩蛋「断桥/油纸伞文化」 → 幕末**抉择②**
5. **第三幕「端午现形」** → 完整叙述饮雄黄酒、白蛇现形、许仙惊吓的全过程 → 触发**物品视角**（金钵的内心独白）→ 触发彩蛋「端午驱五毒民俗」
6. **第四幕「盗仙草」** → 完整呈现白蛇上昆仑求草的冒险 → **抉择③**「法海是否暗中助力？」 → 触发彩蛋「仙草/灵芝神话」
7. **第五幕「水漫金山」**（高潮）→ 完整叙述水斗 → **抉择④（平行时空触发点）**：「坚守天条 / 放下金钵」
   - 选「放下」：3-5 秒短暂幸福画面（白蛇与许仙白头偕老的剪影）→ **画面琉璃破碎特效** → 旁白「然而，这只是金山寺月色下的一个梦」→ 强制回归原典推进
   - 选「坚守」：直接进入水漫金山悲剧高潮
8. **第六幕「断桥重逢」** → 完整叙述产子、断桥重逢 → **视角切换②**（许仙视角）→ 触发彩蛋「警世通言 vs 雷峰塔版本差异」
9. **第七幕「雷峰永镇」** → 完整呈现镇压结局 → **抉择⑤（心境抉择）**：决定法海此刻的心境（坚定/怀疑/觉醒）→ 解锁对应**心境结局**之一（剧情主干一致，结尾旁白与画面氛围不同）
10. **结算页** → 显示本次解锁的结局名 + 文化彩蛋收集进度（如 7/12）+ 还有 X 个结局未解锁 + 「再玩一次」「回到首页」「分享剧情卡片」三个 CTA
11. **分享路径** → 点击「分享剧情卡片」→ 自动生成精美图卡（古画画面 + 关键台词 + 章节名 + 彩蛋收集进度 + 网站二维码）→ 一键保存/分享

**主流程关键说明**：
- **完整叙述**：7 幕一幕不少，每幕都从起因讲到本幕结尾，原典情节完整覆盖
- **抉择贯穿**：5 个关键抉择 + 2 次视角切换 + 1 次物品视角分布在不同幕中，而非堆在结局
- **教育嵌入**：每幕至少 1-2 个文化彩蛋点，玩家可选择是否查看，不打断主线
- **结局忠实**：4 种结局都遵循"白蛇被镇压"的原典结局，只是法海的心境与画面氛围不同；平行时空必定回归原典

### 分支流程

**A. 二刷探索其他视角**：
- 通关后 → 章节卡片显示「已完成 1/4 结局」→ 用户从首页或菜单重进 → 系统自动从分歧点提供「跳过已读」按钮 → 用户做新选择 → 解锁新结局

*B. 探案模式（韩凭/紫玉短篇）**：
- 进入短篇 → 以「说书人/判官」身份引入悲剧现场 → 玩家通过收集信物（书信、玉佩、残魂）触发闪回 → **闪回中完整呈现原典故事**（不是片段，是从相遇 → 发展 → 悲剧的全程） → 全部收集后还原真相
- 真相忠实于古籍记载（《搜神记》原文）—— 揭秘感来自"原来你不知道这段故事"，而非"颠覆民间流传版本"

**C. 视角切换**：
- 阅读中 → 右侧浮出"切换视角"按钮（在多视角节点）→ 点击 → 画面旋转淡出 → 切换到另一个角色视角 → 同一事件用对方眼睛重看 → 看完后回到主视角继续推进

### 出问题了

| 异常情况 | 处理方式 |
|---------|---------|
| 首次进入（无存档） | 直接进入首页，章节卡片标识「未开启」，引导从主推章节开始 |
| 页面加载慢 | Loading 页用国风水墨动画 + 古诗台词（如"金山寺外，妖气浮动……"），加载时间 > 3s 显示进度提示 |
| 用户想中途退出 | 任何时候按 ESC 或点菜单 → 弹出"是否保存进度并退出"确认框 |
| 选择分支后想反悔 | 提供「目录」功能（右下角悬浮控件），可跳回已读过的任何幕重看；但分支选择不可撤销，必须重玩章节 |
| 网络中断 / 资源加载失败 | 显示古卷破损样式的错误提示："此处的故事似乎遗失了……" + 重试按钮 |
| 用户太快点击跳过 | 关键剧情节点（如平行时空破碎）禁用快进，强制完整呈现 |
| 移动端体验 | 触屏上拉推进卷轴；快速上拉 = 阅读加速；上下拉动为主，不做左右滑动 |
| 完整章节未上线（其余母题） | 短篇预告卡片点击 → 显示"章节制作中，敬请期待 + 短篇预告动画"，避免死链 |

## 关键组件细节

### 1. 卷轴文字段系统（原「对话框系统」）

**负责**：推进叙事的核心载体。在数字绘本长卷中，文字不是“点击翻页”出现的，而是随滚轮下拉**逐句淡入**，以类似阅读走马卷的节奏呈现。

| 状态 | 长什么样 | 什么时候触发 |
|------|---------|------------|
| 默认 | 文字以纵向段落呈现于古画上下之间；有言者时顶部有角色名标签 + 头像缩略；进入可视区后逐句淡入 | 随页面滚动 |
| 旁白样式 | 无角色名标签，居中居上出现，字体略放大、色偏暖白 | 叙事者主动叙述 |
| 心声样式 | 字体略偏斜，颜色偏淡，段落背景泛淡墨晕染 | 角色心声独白 |
| 随滚动逐句淡入 | 进入可视区的每句以 200-400ms 淡入 | 页面滚动 |
| 快速下拉 | 用户快速下拉时，未阅句子快速出现不会丢字 | 用户加速阅读 |
| 隐藏控件 | 鼠标静止 1.5秒 / 按 H 隐藏顶部薄条与右下控件，仅保留卷轴主体区 | 用户需专注阅读 / 截图时 |

**明确不做的**：「每句点一下才出下一句」的幻灯片式翻页交互。

### 2. 古画场景系统（原「角色立绘系统」）

**负责**：以传统古画为主体画面，通过柔和的画面过渡与镜头运动营造沉浸感。古画本身是静态的，不加抖动、表情切换、后退等“表演”动作。

| 状态 | 长什么样 | 什么时候触发 |
|------|---------|------------|
| 默认画面 | 全屏呈现一幅古画（如《雷峰塔传》插画、敦煌壁画、木版年画），顶部/底部留有足够留白放置文字与控件 | 进入场景 |
| 画面切换 | 两幅古画之间以「交叉淡入淡出」或「卷轴展开」过渡，800-1200ms 柔和换帧 | 剧情推进/场景转换 |
| 镜头推进 | 同一幅古画上缓慢 Ken Burns（轻微缩放 + 平移），引导视线 | 重点人物/物件上场时 |
| 局部高亮 | 在古画上叠加柔光晕圈或轻薄蒙层，突出当前讲述的人物/物件 | 当前发言者/重点讲述对象 |
| 画面叠交 | 文化彩蛋浮窗出现时，背景古画轻微变暗不模糊 | 触发彩蛋时 |
| 纯文字画面 | 仅背景为纯色/宣纸背景 + 居中旁白文字 | 开场/转场旁白、独白独立场景 |

**明确不做的动作**：古画抖动、表情切换、人物后退、转身、跳跃、动态微表情。这些动作与古画的静态质感冲突，也与“古老叙事”的气质不符。

### 3. 叙事抉择浮层（原「选择分支浮层」）

**负责**：让用户选择「从何种视角/心境看这个故事」。**抉择不改变事件本身，只改变后续叙事的视角/旁白/氛围**。

| 状态 | 长什么样 | 什么时候触发 |
|------|---------|------------|
| 浮现 | 画面轻微变暗（20% 蒙层）+ 2-3 个选项卡片从屏幕中间向上浮出，竹简/卷轴样式 | 叙事节点 |
| 抉择提示语 | 选项句式以「你现在想 … / 你选择从 … 看」起头，强调“视角选择”而非“行为选择”（如「你选择从法海的角度看下去 / 你选择从白蛇的角度看下去」） | 选项文案 |
| 悬停 | 选项卡片轻微放大 + 描边柔光 + 出现“预览”小字（如「这一段将以法海的心境为主」） | 鼠标悬停 |
| 点击 | 选项卡片柔和收拢 + 画面交叉淡入 → 后续同一事件以不同视角/旁白呈现 | 用户选择 |
| 未选提醒 | 文案透露「另一个视角可重读体验」，鼓励重读而非二刷 | 选择后提示 |

**设计原则**：选项背后的“故事”完全一致，只是“讲述者的身份”不同。避免让用户产生“我是不是选错了”的游戏化焦虑。

### 4. 视角切换组件

**负责**：实现"反差叙事"——把熟知的故事换个滤镜重看。

| 状态 | 长什么样 | 什么时候触发 |
|------|---------|------------|
| 切换提示 | 屏幕右侧浮出可切换角色的头像按钮 + 「切换视角」标签，柔光呼吸动画 | 多视角节点 |
| 切换中 | 画面以"翻书/卷轴展开"动画过渡 → 镜头转向另一角色 | 用户点击切换 |
| 视角标识 | 章节顶部固定显示"当前视角：法海"标签（可隐藏） | 切换后持续显示 |
| 同事件复述 | 重新播放同一场景，但对白、立绘表情、内心独白全换；关键道具会高亮呈现"原来这才是真相" | 切换视角后 |

### 5. 信物探案系统

**负责**：冷门母题（韩凭、紫玉等）的核心玩法。

| 状态 | 长什么样 | 什么时候触发 |
|------|---------|------------|
| 场景探索模式 | 鼠标变成放大镜，可点击元素带柔光呼吸 | 进入探案章节 |
| 信物未发现 | 元素仅有微弱呼吸光（盲点） | 默认状态 |
| 信物悬停 | 光晕加强 + 显示物品名（如「染血的罗帕」） | 鼠标悬停 |
| 信物收集 | 点击 → 物品旋转浮起 → 进入档案册 → 触发短闪回视觉小说（30-60 秒）| 用户点击 |
| 档案册 | 右下角小图标，点击展开当前章节已收集信物 + 关联线索 | 玩家查阅 |
| 拼凑真相 | 全部信物收集后，触发"还原真相"动画 → 玩家选择如何讲述真相 | 探索完成 |

### 6. 「如果当时」幻象浮窗（原「平行时空 / 结局破碎特效」）

**负责**：在原典高潮点，以**幻象/梦境**形式呈现一段“如果不是这样”的诗意画面，随即回归原典。**本质上这是一个嵌在原典中的「诗意插话」，而非“可选的另一种结局”**。

| 状态 | 长什么样 | 什么时候触发 |
|------|---------|------------|
| 进入幻象 | 画面色调柔和变暖（金/粉调）+ BGM 转柔和古琴 + 旁白「后世画师笔下的他们，也曾有过这样一幅 … 」或「民间口耳相传里，也留下过这样一种版本 … 」 | 原典高潮点另起一段 |
| 古画定格 | 3-5 秒定格一幅「幻象古画」（如仿《踏歌图》风格的“白蛇与许仙白头偕老”插画），可阅读一句诗意旁白 | 幻象持续期 |
| 画面消散 | 画面边缘柔和洇散/墨迹晕开，如古画逐渐被水润褪色，取代琉璃破碎 | 幻象结束的瞬间 |
| 回归原典 | 画面消散后重新呈现原典高潮古画，BGM 切回原调，旁白「然而，原典中记载的，是另一个结局。」→ 正常推进原典高潮 | 消散完成 |

**设计原则**：这不是“用户选了个平行结局”，而是“叙事者给用户看一眼后世另一种叙事可能”，随即回到原典。避免“如果选得好就能改写结局”的游戏化误解。触发不需要用户选择，是作为剧情中的一个叙事插话点自然出现。

### 7. 文化彩蛋 / 文献注解系统

**负责**：在沉浸叙事中保留教育意义，让玩家"顺便学到东西"，触发"原来如此"。

| 状态 | 长什么样 | 什么时候触发 |
|------|---------|------------|
| 隐藏入口 | 场景中的关键道具/地名/人物附带柔和呼吸光（不强制点击） | 默认 |
| 悬停提示 | 鼠标悬停 → 浮出小标签（如「金钵」、「断桥」、「雄黄酒」） | 鼠标悬停 |
| 展开浮窗 | 点击 → 古籍卷轴样式浮窗弹出，包含三块内容：　① 这是什么（概念解释）　② 文献出处（如《雷峰塔》第几出、《搜神记》原文摘录）　③ 文化背景（民俗/历史小知识） | 用户点击 |
| 关闭 | 点击空白处或浮窗关闭按钮 → 浮窗淡出，不打断剧情 | 用户主动关闭 |
| 收集进度 | 章节内已点开的彩蛋会变金边并计入「文化收集册」 | 自动 |
| 全收集奖励 | 一章彩蛋全部收集 → 解锁「学究」称号 + 专属分享卡片模板 | 全部收集后 |

**触发铺设密度**：每章 8-12 个彩蛋点，分布在每一幕，平均每 2-3 分钟剧情可遇 1 个。

### 8. 多结局收集墙

**负责**：通关后驱动二刷的成就系统。

| 状态 | 长什么样 | 什么时候触发 |
|------|---------|------------|
| 未解锁结局 | 灰色剪影 + ?号 + 「未发现」标签 | 默认 |
| 已解锁结局 | 完整结局插图 + 结局名 + 解锁日期 + 「关键选择回顾」按钮 | 通关后追加 |
| 全收集 | 显示一个隐藏的"作者寄语"卡片 + 解锁特殊分享卡片 | 全部结局解锁 |
| 分享 | 每个已解锁结局都可单独生成精美分享卡片 | 用户点击分享 |

### 9. 剧情卡片分享生成器

**负责**：把"我玩到了某个剧情"转化为可发到小红书/微博/朋友圈的社交货币。

| 状态 | 长什么样 | 什么时候触发 |
|------|---------|------------|
| 触发入口 | 章节内任何时候按相机按钮，或结局结算页"分享"按钮 | 用户主动 |
| 卡片预览 | 弹窗显示生成中的卡片：当前角色立绘 + 关键台词 + 章节名 + 章节进度 + 网站二维码，可在 3-4 个模板间切换 | 触发后 |
| 卡片下载 | 「保存到本地」按钮 → 高清 PNG 落地 | 用户保存 |
| 一键转发 | 「分享到小红书 / 微博」按钮 → 调用对应 SDK 或复制图文到剪贴板 | 用户分享 |
| 自定义台词 | 高级用户可编辑卡片上的台词（限定字数） | 二级操作 |

### 10. 顶部薄条 + 右下角悬浮控件（原「轻量控制栏」）

**负责**：卷轴阅读体验下的辅助控件。为了不遮挡卷轴主体区，所有控件采用**顶部薄条 + 右下角悬浮**两部位分散安放，鼠标静止后自动隐藏。阅读进度通过浏览器 localStorage 自动保存。

| 位置 | 功能 | 行为 |
|------|------|------|
| 顶部薄条 | 返回、章节名、幕名、进度 ％ | 随页面滚动半透明吸顶；鼠标静止 1.5秒隐藏 |
| 顶部薄条 · 进度 | 以细线呈现当前幕位置（如 38%），可点击快速跳到本幕某点 | 让用户知道卷轴还有多长 |
| 顶部薄条 · 返回 | 返回首页 | 随时退出，进度自动保存 |
| **右下·自动播放（Auto）** | 开启后页面按 1.5-4 秒/句的节奏自动下拉；拦轴点自动暂停等待用户 | 古画+文字阅读型体验下，自动播放使用户能专注欣赏画面 |
| 右下·目录 | 弹出幕表面板，可跳到已读过的任何幕 | 代替传统「回看」，适合长卷阅读 |
| 右下·文化收集册 | 已收集的文化彩蛋小档案 | 强化“玩中学”的获得感 |
| 右下·设置 | 文字淡入速度 / 自动播放间隔 / BGM 音量 / 音效音量 | 仅保留最必要的项 |
| 右下·退出 | 退出当前章节，进度自动保存到浏览器 | 无需手动存档 |
| 隐藏逻辑 | 鼠标静止 1.5秒 / 按 H 隐藏顶部薄条与右下控件 | 保障专注阅读与截图体验 |
| ❌ 不做 | 快进 / 多存档位 / 读档 / 成就 / 好感度 / 底部固定控制栏 | 这些与沉浸阅读型长卷体验冲突 |

## 附录 B：交互节奏建议

| 节奏维度 | 建议 |
|---------|------|
| 卷轴总长（高度） | 单幕约 8-12 屏高（1 屏 ≈ 1280-1440px），全章 50-80 屏高 |
| 卷上古画密度 | 每幕 4-6 幅古画为主调铺设，每幅古画后跟 1-3 段卷轴文字 |
| 文字淡入节奏 | 每句进入可视区后 200-400ms 淡入，句与句间隔 0-200ms |
| 自动下拉节奏 | 开启 Auto 后，默认 1.5-4 秒/句；拦轴点自动暂停 |
| 卷上亮点密度 | 每 2-3 屏高嵌 1 个，每幕 4-8 个 |
| 卷中拦轴密度 | 全章 5 个抉择拦轴 + 2 个视角切换拦轴，平均每幕 1 个 |
| 视角切换密度 | 每章 1-2 次，分布在中段（不放在开场，避免削弱原典代入） |
| 文化彩蛋密度 | 每章 8-12 个彩蛋点，每幕至少 1-2 个，玩家自主选择是否查看 |
